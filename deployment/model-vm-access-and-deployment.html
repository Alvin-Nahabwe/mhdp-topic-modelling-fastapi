MHDP Model VM Access & Deployment Guide

1. Overview

This guide explains how to:

  • Join the secure network (Tailscale)
  • Access the Model VM via SSH
  • Deploy services using GitHub Actions

Note: The Model VM is not publicly exposed and is only accessible via Tailscale.

2. Prerequisites

  • Tailscale installed
  • Access to Tailnet (invitation link will be provided)
  • SSH private key
  • GitHub repository access

3. Join the Tailscale Network

Step 1: Install Tailscale

Download and install from the official website.

Step 2: Join Tailnet

  • Open the invitation link (to be provided)
  • Log in and accept access

Step 3: Verify connection

tailscale status

4. Obtain Model VM IP

  • Open Tailscale dashboard
  • Locate Model VM
  • Copy IP (format: 100.x.x.x)

5. SSH Access

Command:

ssh -i "<path-to-private-key>" inference@<tailscale-ip>

Example:

ssh -i "C:\keys\inference_id_rsa" inference@100.xxx.xxx.xxx
Notes:
  • Username: inference
  • SSH Key-based authentication only

6. Deployment via GitHub Actions

GitHub Actions connects to the Model VM through Tailscale before deployment.

Tailscale Step

- name: Tailscale 
  uses: tailscale/github-action@v4
  with:
    authkey: ${{ secrets.TAILSCALE_AUTHKEY }}
Note: The TAILSCALE_AUTHKEY will be provided securely and needs to be added to GitHub Secrets.

Deployment Flow

  • Build Docker images
  • Push to registry
  • Connect to Tailnet
  • SSH into Model VM
  • Deploy using Docker Compose
All communication occurs securely over Tailscale. No public IP is used.

7. Running sudo Commands

The inference user has permission to run administrative commands using sudo.

Example:

sudo docker ps
Notes:
  • Sudo commands require a password for execution
  • The password will be provided securely
  • You will be prompted to enter the password when running a sudo command

8. Security Overview

  • No public exposure
  • SSH key-based authentication only
  • Firewall restricted to Tailscale network
  • User roles:
    • inference → deployment

9. Troubleshooting

SSH Issues:

  • Ensure Tailscale is connected
  • Verify correct IP
  • Check SSH key permissions

GitHub Actions Issues:

  • Verify TAILSCALE_AUTHKEY
  • Ensure Tailscale step is included
  • Confirm VM is online

10. Summary

  • Access via Tailscale only
  • SSH is secure and key-based
  • Deployment is automated via GitHub Actions
